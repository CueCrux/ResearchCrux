## Chain Verification

Receipts form an append-only linked list via `parentSnapId`. Chain verification traverses the list using a recursive CTE (depth limit 50). Verified performance: 2–4ms regardless of depth.

For independent verification without CueCrux infrastructure, see the [verification walkthrough](../../proof-gallery/redacted/verification-walkthrough.md).

## Signing

Receipts are signed with **ed25519** via HashiCorp Vault Transit. Each receipt carries its public key (`signature.pubB64`), enabling verification without access to the signing infrastructure. Key rotation is transparent — old receipts remain verifiable.
